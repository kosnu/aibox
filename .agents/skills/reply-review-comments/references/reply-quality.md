# Reply Quality

Write every actual GitHub reply in Japanese, even when examples, sample replies, review comments, code, branch names, or commit messages are in English. English samples may remain English; sample language is not output-language guidance.

For each reply:

- explain the concrete handling in 1-3 Japanese sentences
- do not reply with only "fixed" or "addressed"
- mention files or functions only when they help the reviewer
- include a commit ID only when the change is committed and attribution is clear
- do not wrap commit IDs in backticks
- put one ASCII space before and after a commit ID in prose
- do not invent a commit ID or code change

Resolve a thread only when all conditions hold:

- the requested issue is fixed on the current branch
- no follow-up work or confirmation is pending
- the reply matches the implementation
- no conflicting comment or unresolved concern remains

Do not resolve partially handled threads.
